Etiquetas Semanticas Estructurales

DIV: Es una etiqueta contenedora que no tiene semantica. El problema es q si no teenes bien identado el codigo te confundis cuando abre una y cuando cierra otra. Antes de usar una div intentar usar otra etiqueta especializada para lo que quiera escribir. Por esto, luego nacen nuevas etiquetas:
HEADER: Esta etiqueta nos representa la cabecera de un sitio o seccion
MAIN: nos va a definir la seccion principal del documento. Solo puede existir una etiqueta main por documento. Igual que el h1. Te dice "aca esta el contenido principal"
FOOTER: pie de pagina de un sitio web o de una seccion
ARTICLE: es un articulo. representa una seccion autocontenido que por si sola se explica sin tener contexto de toda la interfaz del sitio.
ASIDE: representa contenido complementario o secundario. Tipica lista de "articulos relacionados". Tambien sirve para las zonas de publicadad de los sitios web.
SECTION: representa una seccion de contenido generico. Se discute si un article tiene que ir dentro de un section o viceversa, la verdad es q la semantica es subjetivo en base al tipo d contenido q definas en tu web. Podemos tener seccion que tenga su propio articulo con una seccion interna. No es a rajatabla.
NAV: esta etiqueta es una para navegacion. Suelen tener links donde clickeas y te envian a otra web
ADDRESS: representa una direccion fisica.

Etiquetas de bloque vs. etiquetas de linea

De todas las etiquetas de html hay un conjunto q trabaja en linea y hay otro que trabaja en bloque.

La etiqueta de bloque por excelencia es la DIV que no tiene semantica. Una etiqueta de bloque es aquellas que genera saltos de linea con sus etiquetas hermanas. Van a generar un enter o salto de linea tanto arriba como abajo. Una etiqueta de bloque usa todo el ancho disponible.
Desde html podemos aplicar estilos desde una etiqueta con el atributo STYLE. Una etiqueta de bloque usa todo el ancho disponible. Prueba de eso es que el color continua incluso luego de q termina el texto hasta ocupar todo el largo o ancho.
La etiqueta de linea por excelencia es SPAN que no tiene semantica. Una etiqueta de linea solo ocupa el espacio necesario que tiene su contenido. Prueba de esto es que el color azul solo ocupa el texto y no continua ocupando el ancho de todo el bloque. Hola, soy otra SPAN. Como solo ocupa el espacio que pretende y no un bloque, esto puedo continuar sin ocupar un espacio por si mismo.

Los encabezados son una etiqueta de bloque ya que ocupan todo un bloque
